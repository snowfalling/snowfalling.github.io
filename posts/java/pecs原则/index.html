PECS原则

JAVA的PCES理解
2026年6月25日 · 1 分钟 · 152 字 · Ronald

PECS 是 Java 泛型中的一个实用准则，全称是 Producer Extends, Consumer Super，由 Joshua Bloch 在《Effective Java》中提出。核心思想是：如果一个参数化类型是数据的生产者（你从中读取），用 ? extends T；如果是数据的消费者（你往里写入），用 ? super T。

为什么需要它？

Java 的泛型是不变的（invariant）：List<Dog> 不是 List<Animal> 的子类型，即使 Dog extends Animal。通配符是打破这个限制的方式，而 PECS 告诉你该选哪种通配符。

Producer Extends（读取用 extends）

当你只需要从集合中取出元素时：

public double sum(List<? extends Number> list) {
    double s = 0;
    for (Number n : list) s += n.doubleValue(); // 读取，OK
    // list.add(1);  // 编译错误！不能写入
    return s;
}

这样 List<Integer>、List<Double> 都能传进来。编译器知道取出的一定是 Number 的子类，所以读取安全；但不知道具体是哪个子类，所以禁止写入。

Consumer Super（写入用 super）

当你只需要往集合中放入元素时：

1
2
3
4
5

public void fill(List<? super Integer> list) {
    list.add(1);    // 写入，OK
    list.add(2);
    // Integer x = list.get(0);  // 编译错误！取出只能当 Object
}

List<Integer>、List<Number>、List<Object> 都能传进来。编译器知道容器至少能装 Integer，所以写入安全；但取出时只能保证是 Object。

一个典型例子：Collections.copy

1
2
3
4

public static <T> void copy(List<? super T> dest, List<? extends T> src) {
    for (int i = 0; i < src.size(); i++)
        dest.set(i, src.get(i));  // 从 src 读，往 dest 写
}

src 是生产者（读） → extends，dest 是消费者（写） → super。

**简单记法：**往外拿东西的用 extends，往里塞东西的用 super，又拿又塞就用精确类型 T，不用通配符。
