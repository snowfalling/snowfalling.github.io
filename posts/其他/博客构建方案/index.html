使用 Hugo + Obsidian + GitHub Pages 搭建个人博客

为什么选择这套方案#

在选择博客方案时，我有几个核心需求：

  1. 用 Obsidian 写作 — 日常笔记和博客统一管理，不想切换工具
  2. 私有内容保护 — 有些笔记是私密的，不能发布到公网
  3. 自动化部署 — 推送即发布，不想手动操作
  4. 免费托管 — GitHub Pages 完美满足

最终方案：Obsidian 写作 → GitHub 私有仓库 → Hugo 构建 → GitHub Pages 部署。

两个仓库各司其职：

  • Blog (私有) — 存放 Obsidian 原始 Markdown 文件
  • snowfalling.github.io (公开) — 只存放 Hugo 生成的静态 HTML

baseURL = "https://snowfalling.github.io/"

title = "Ronald's Blog"
theme = "PaperMod"

# 忽略 private 文件夹
ignoreFiles = ['content/private/']

# 允许 HTML（Obsidian 笔记可能包含 HTML）
[markup.goldmark.renderer]
  unsafe = true

3. 处理 Obsidian 兼容性#

Obsidian 使用 [[双链]] 和 ![[图片]] 语法，Hugo 不认识。写了 scripts/sync.sh 自动转换：

  • [[页面名称]] → [页面名称](/posts/页面名称/)
  • [[页面|显示文本]] → [显示文本](/posts/页面/)
  • ![[image.png]] → ![image](/images/image.png)

4. 私有内容过滤#

通过两层机制保护隐私：

  1. 目录级别 — content/private/ 整个目录在构建时删除
  2. 文件级别 — front matter 中 publish: false 的文章不会发布

scripts/filter.sh 在 CI 构建前执行，确保私密内容不会出现在最终站点中。

5. GitHub Actions 自动部署#

配置 .github/workflows/hugo_update.yml，push 到 master 分支自动触发：

  1. Checkout 代码（含 submodule）
  2. 安装 Hugo
  3. 运行 sync.sh（Obsidian 格式转换）
  4. 运行 filter.sh（过滤私有内容）
  5. Hugo 构建
  6. 部署到 snowfalling.github.io

6. 部署密钥配置#

# 生成 SSH 密钥对

总结#

这套方案实现了我想要的所有能力：Obsidian 统一管理笔记和博客，私有内容安全隔离，push 即发布零操作。整个搭建过程大约半小时，后续只需要专注写作即可。
